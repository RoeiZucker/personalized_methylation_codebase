# Evaluation

## Purpose

Evaluate the retrained models across the checkpoints generated by the training workflow.

## Main flow

- Create eval YAMLs for `no_pretraining` retrain outputs.
- Create eval YAMLs for retrains that start from pretrain checkpoints.
- Run the evaluation script on each eval YAML.
- Write result CSV files under the project results directory.

## Inputs

- Retrain test dataset.
- Variability CSV.
- Model checkpoint paths collected from training outputs.

## LoRA variant

- Evaluation also covers the LoRA retrain variants.
- It follows the same shape as regular evaluation with different source model paths.
